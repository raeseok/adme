# AdMe 카카오톡 광고 도착 알림 — 가능성 검토 (Stage 2-A)

작성일: 2026-07-08  
목적: 소비자 맞춤 광고 도착 시 카카오톡 알림 기능의 구현 가능성·정책·데이터·Stage 분해 정리  
**Stage 2-A에서는 실제 구현·발송 없음**

---

## 1. 기능 개요

- **정의**: 소비자에게 맞춤 소비정보/광고 카드가 도착했음을 카카오톡으로 알려주는 기능
- **앱 내 처리**: 광고 본문 확인, 최소 열람, 퀴즈 응답, 서버 채점은 **AdMe 앱 내부**에서만 수행
- **카카오톡 역할**: 앱 진입 유도용 알림으로 제한 — 퀴즈·정답·포인트 적립은 AdMe 앱 내부 서버 권위 흐름으로만 처리

---

## 2. 구현 가능성 판단

| 항목 | 판단 |
|---|---|
| 기술적 구현 가능성 | **가능** |
| Stage 2-A 실제 발송 | **하지 않음** |
| Kakao Developers 일반 Talk Message API 직접 채택 | **하지 않음** — 소비자 대상 자동 광고 알림 수단으로 부적합 |
| 향후 검토 방향 | 카카오톡 채널 / 비즈메시지 / 브랜드 메시지 / 친구톡 계열 또는 적법한 비즈니스 메시지 발송 체계 |

---

## 3. 알림톡 적용성 판단

- 알림톡은 주문·결제·배송 등 **정보성 메시지** 중심
- "새 광고 도착" 알림에는 **부적합**하거나 **심사 리스크가 높을** 수 있음
- AdMe 메시지는 소비자가 사전에 요청한 소비정보라는 사업 철학을 가지나, 카카오 정책상 **광고성 메시지**로 판단될 가능성이 높음

---

## 4. 수신 동의 원칙

- 카카오톡 알림 **기본값 ON 금지**
- 소비자가 명시적으로 **"카카오톡으로 맞춤 소비정보 도착 알림 받기"** 선택 필요
- **광고성 정보 수신 동의** 및 **수신거부** 기능 필요
- 수신거부는 앱 내 알림설정에서 언제든 가능해야 함

---

## 5. 메시지 내용 원칙

카카오톡 메시지에 **포함하지 않음**:

- 퀴즈 정답, 정답 추론 정보, 정답 option id
- `point_ledger` 결과, 적립 완료 문구

메시지는 **"새 소비정보 도착 → 앱에서 확인"** 수준으로 제한.

### 예시 문구 (확정 템플릿 아님)

```
(광고) AdMe
관심 지역·분야에 맞는 새 소비정보가 도착했습니다.
앱에서 광고 내용을 확인하고 퀴즈에 참여해 보세요.
수신거부: AdMe 알림설정
```

---

## 6. 개인정보 분리 원칙

- 소비 의향 프로필(`consumer_profiles`)은 개인 신원 정보가 아니므로, 카카오톡 발송 연락수단을 **섞지 않음**
- 전화번호, 카카오 채널 식별자, 마케팅 수신 동의 이력은 **별도 알림 테이블**로 분리
- 광고주는 소비자 연락수단을 **절대 볼 수 없음** — 집계 통계만 제공

---

## 7. 향후 테이블 후보 (Stage 2-A에서 DB 생성 없음)

| 테이블 | 역할 |
|---|---|
| `notification_preferences` | 채널별 수신 동의, 광고성 정보 동의, 수신거부 |
| `notification_contact_points` | 카카오 발송 식별자 (암호화·별도 보호) |
| `notification_events` | 매칭 광고 발생 시 알림 후보 이벤트 |
| `notification_deliveries` | 발송 성공/실패/차단/재시도 기록 |

---

## 8. 개발 Stage 반영안

| Stage | 범위 |
|---|---|
| **2-A** | read-only 광고 카드·퀴즈 UI + 본 문서화. **카카오톡 실제 발송 없음** |
| **2-B** | 최소 열람 타이머 + 서버 채점. 카카오 발송 없음. point_ledger mutation 없음 |
| **2-C** | ad_views attempt + 재도전. 카카오 발송 없음. point_ledger mutation 없음 |
| **2-D** | 알림 선호 설정 UI + 인앱 알림 큐 + 카카오톡 수신 동의 상태 저장. **실제 카카오 발송 없음** |
| **5-K** | 카카오톡 실제 발송 연동·파일럿 검증 |

---

## 9. Stage 2-A 금지 범위

- 카카오 API 호출 금지
- Kakao access/refresh token 저장 금지
- 휴대전화번호 수집 금지
- 카카오톡 채널 친구 여부 조회 금지
- 실제 발송 금지
- 발송 대행사 API 연동 금지
- message template 등록/심사 자동화 금지

---

## 10. 운영 리스크

- 광고성 정보 수신 동의 필요
- 수신거부 처리 필요
- 야간 발송 제한 등 관련 법령·정책 검토 필요
- 카카오 정책/상품 변경 가능성
- 개인정보 분리 보관 필요
- **개발/운영 Supabase 미분리** 상태에서는 실제 연락수단 수집을 시작하면 안 됨

---

## 11. 핵심 결론

카카오톡 광고 도착 알림은 **구현 가능**하나, Stage 2-A에서는 **feasibility 문서와 로드맵 반영만** 수행한다.  
실제 발송은 **수신 동의·연락수단 분리·dev/prod 분리** 후 Stage 2-D(동의 저장) → Stage 5-K(실발송)에서 진행한다.
